## 🔴 CRITICAL BUG DISCOVERED — Race Condition in Paper Trading

**Symptom:** Paper trading balance corrupted to `$756,429.70` (from $10,000), PEPE position size `$37,821` (should be ~$500).

**Root Cause:** Position monitor and auto-executor were both reading/writing `logs/paper_trading.json` simultaneously without proper coordination. Race condition caused:
- Outdated balance read by one process
- Double-counted trades
- Multiplied position sizes

**Fix Applied:**
- Stopped all trading sessions
- Reset `paper_trading.json` to clean state ($10,000, 0 trades)
- Position monitor now writes portfolio summary to separate file (not modifying paper_trading.json)
- `file_lock.py` exists with `safe_read_json()` / `safe_write_json()` using `fcntl` file locking
- Auto mode set to OFF for safety during verification

**Restart:** 8 core sessions running (without position-monitor and auto-executor until fully verified).
